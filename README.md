# Price Gap Finder MVP

Yahooショッピングの商品仕入れ価格と、買取Wikiの買取価格を比較して、利益が出る候補を見つけるためのブラウザ完結MVPです。

## できること

- 商品の `purchasePrice` と `buybackPrice` を JSON で投入する
- 送料と手数料率を加味して純利益を計算する
- 最低利益額と最低利益率で候補をフィルタする
- 利益が高い順に並べて、仕入れ候補を優先表示する

## 使い方

1. `index.html` をブラウザで開く
2. 「サンプルデータを読み込む」で画面動作を確認する
3. `sourceInput` に自分のデータを JSON 配列で貼り付ける
4. しきい値を調整して「利益候補を再計算」を押す

## 今後の拡張方針

### 1. データ取得レイヤーを分離する

以下のように取得元ごとにモジュールを分けると、MVPから実運用へ伸ばしやすいです。

- `connectors/yahoo.ts`
- `connectors/kaitoriWiki.ts`
- `core/profitCalculator.ts`
- `storage/history.ts`
- `notifications/slack.ts`

### 2. 注意点

- Yahooショッピングや買取Wikiの取得方法は、各サービスの利用規約や robots、API利用条件を確認してから実装してください。
- 実運用では `在庫切れ`、`中古/新品の違い`、`色違い`、`型番のブレ` を吸収する正規化処理が重要です。
- 利益判定には `ポイント還元`、`クーポン`、`振込手数料`、`キャンセル率` も加味すると精度が上がります。

### 3. おすすめの次ステップ

- まずは手動またはCSV取込で10〜30件の検証を回す
- 利益判定ロジックが固まったら自動取得を足す
- 最後に通知と履歴分析を追加して、張り付き確認を減らす
